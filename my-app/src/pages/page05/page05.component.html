  1. animateChild 与 query: animateChild 必须要与 query 配合使用，但是 query 单独使用也能做到触发子动画。所以显得 animateChild 有些多余。 在测试过程中发现，当子动画使用 trigger 定义的指令时，此时运行动画，只有父动画在运行，子动画不会运行。需要两个函数搭配使用才能使子动画运行。然而，如果子动画是使用 query 定义的，这时候父动画与子动画都可以运行。 个人理解：我猜测，当动画指令可以复用时，两者搭配使用更简洁。避免了每次都使用 query 单独定义相同的动画。
  2. stagger: 该函数依然需要与 query 搭配使用。需要注意的是，stagger 设置的延时并不是针对两个 query 函数之间，而是一个 query 函数查询出的元素数组之间会延时执行。比如：query('div', ...)可以查询到三个 div ，那么这三个div动画循序执行之间会有延时。
